# 受講生席替えアプリ (Streamlit + SQLite)

受講生データ管理、席替え自動生成、手動微調整、CSV/Excel/Googleスプレッドシート出力、履歴管理を1つの画面で行うアプリです。

## 1. 動作環境

- Python 3.11 以上推奨
- `pip` が利用可能
- ブラウザで `http://127.0.0.1:8501` にアクセス可能

## 2. セットアップ

プロジェクトルート（`app.py` がある場所）で実行してください。

macOS / Linux:

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -r requirements.txt
```

Windows (PowerShell):

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
```

## 3. 起動と停止

起動:

```bash
python -m streamlit run app.py
```

ポート変更:

```bash
python -m streamlit run app.py --server.port 8502
```

停止: 実行中ターミナルで `Ctrl + C`

## 4. 画面構成

タブ構成:

- `受講生管理`
- `席替え実行`
- `席替え結果`
- `履歴`

### 4-1. 受講生管理

主な機能:

- 受講生の追加 / 編集 / 削除
- CSV取り込み（追記 / 全置換）
- スキル一括更新
- ダミー73名投入

CSV取り込み仕様:

- 必須列: `name`, `company`
- 任意列: `name_kana`（ふりがな）, `skill_level`
- `skill_level` が空欄なら `中` 扱いで取り込み
- `name_kana` ヘッダは次も対応: `かな`, `カナ`, `ふりがな`, `フリガナ`, `氏名かな`, `氏名ふりがな` など
- 既存の同一受講生（氏名+会社）がいても、CSV側にふりがながあれば補完更新

スキル表記:

- 画面表示: `高`, `中`, `やや低`, `低`
- 互換入力（CSV/一括更新）: `高い`, `並`, `低い`, `ヤバい` も受け付け

スキル一括更新仕様:

- 受講生名は変更せずスキルのみ更新
- 入力形式:
  - `受講生<TAB>スキル`（Excel 2列貼り付け）
  - `受講生=スキル`
  - `受講生:スキル`
  - `受講生,スキル`

### 4-2. 席替え実行

設定項目:

- `使用テーブル数`
- `1テーブル最小人数`
- `1テーブル最大人数`
- 重み:
  - 同じ会社回避の強さ
  - 前回同席回避の強さ
  - 同スキル固めの強さ
  - 人数均等化の強さ
  - ランダム性
- `試行回数`
- `乱数シード（任意）`
- `対象週`

スキル制約（現行）:

- 禁止同席（ハード制約）:
  - `高 × やや低`
  - `高 × 低`
- 回避優先（ソフト制約）:
  - `中 × 低` は可能な限り回避
- 優先混在:
  - `高-中`, `中-やや低`, `やや低-低`

### 4-3. 席替え結果

主な機能:

- 結果確認
- 手動調整（`table_no` 変更）
- テーブル配置設定（前4・中5・後4）
- CSV/Excel/Googleスプレッドシート出力
- 履歴保存

テーブル配置設定:

- 前列: 最大4テーブル
- 中列: 最大5テーブル
- 後列: 最大4テーブル
- 1〜テーブル数を重複なしで指定
- 未指定テーブルがあるとエラー

出力仕様:

- CSV出力:
  - `table_no, student_id, name, name_kana, company, skill_level ...`
  - `skill_level` は表示ラベル（`高/中/やや低/低`）で出力
- Excel出力:
  - 2シート出力（受講生視点 / 講師視点）
  - スキルは出力せず、`ふりがな・氏名・会社` のみ
- Googleスプレッドシート出力:
  - 2シート作成（受講生視点 / 講師視点）
  - スキルは出力せず、`ふりがな・氏名・会社` のみ
  - 講師視点シートは、受講生視点シートの `B5:T18` を 180°反転
  - 反転時に値・書式・結合セル・行高・列幅も反転配置

出力名ルール:

- 入力名を `○月○日座席表` とした場合:
  - 通常: `○月○日座席表（受講生視点）`
  - 反転: `○月○日座席表（講師視点）`

### 4-4. 履歴

主な機能:

- 過去結果の確認
- 過去結果CSV出力
- 過去結果を結果タブに再読込

## 5. Googleスプレッドシート自動出力の準備

1. Google Cloudでサービスアカウントを作成
2. JSONキーを発行してダウンロード
3. JSON内 `client_email` を対象スプレッドシートへ編集者共有
4. 結果タブで以下を入力
   - 出力先スプレッドシートURL/ID
   - テンプレートシート名（例: `座席表（テンプレ）`）
   - サービスアカウントJSON
5. `Googleスプレッドシートへ2種類を作成` を実行

補足:

- 指定テンプレート名が見つからない場合は `座席表（テンプレ）` やテンプレ相当シートへフォールバックします。

## 6. トラブルシュート

- `streamlit` が見つからない
  - 仮想環境を有効化後、`python -m pip install -r requirements.txt`
- `Port 8501 is not available`
  - 既存プロセス停止、または `--server.port 8502`
- Google出力で失敗
  - JSONが正しいか
  - `client_email` が対象シートの編集者か
  - テンプレートシート名が正しいか
