# 受講生席替えアプリ (Streamlit + SQLite)

毎週の席替え運用向けアプリです。  
受講生データ管理、席替え自動生成、手動微調整、CSV/Excel/Googleスプレッドシート出力、履歴管理までを1つの画面で行えます。

## 1. 動作環境

- Python 3.11 以上推奨
- `pip` が使えること
- ブラウザで `http://127.0.0.1:8501` にアクセスできること

## 2. セットアップ

プロジェクトルート（`app.py` がある場所）で実行してください。

macOS / Linux:

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -r requirements.txt
```

Windows (PowerShell):

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
```

## 3. 起動と停止

起動:

```bash
python -m streamlit run app.py
```

起動後、ブラウザで `http://127.0.0.1:8501` を開きます。  
ポート変更する場合:

```bash
python -m streamlit run app.py --server.port 8502
```

停止: 実行中ターミナルで `Ctrl + C`

## 4. 画面構成（タブ別）

アプリは以下の4タブ構成です。

- `受講生管理`
- `席替え実行`
- `席替え結果`
- `履歴`

---

### 4-1. 受講生管理

主な用途:

- 受講生の追加 / 編集 / 削除
- CSV取り込み（追記 or 全置換）
- スキル一括更新
- ダミー73名投入

操作ポイント:

1. `検索（氏名 / 会社名）` で絞り込み
2. `受講生追加` フォームで単体追加
3. 一覧行ごとに `編集` / `削除`
4. CSV取り込み:
   - 必須列: `name,company,skill_level`
   - 任意列: `name_kana`（氏名ふりがな）
   - `追記` または `全置換` を選択
5. スキル一括更新（名前は変更せずスキルだけ更新）
   - 形式例:
     - `受講生<TAB>スキル`（Excel 2列貼り付け）
     - `受講生=スキル`
     - `受講生:スキル`
     - `受講生,スキル`

`skill_level` 許可値:

- `高い`
- `並`
- `低い`
- `ヤバい`

---

### 4-2. 席替え実行

主な用途:

- 自動席替えの条件を設定して実行

設定項目:

- `テーブル数`
- `1テーブル最小人数`
- `1テーブル最大人数`
- 重み:
  - 同じ会社回避の強さ
  - 前回同席回避の強さ
  - 同スキル固めの強さ
  - 人数均等化の強さ
  - ランダム性
- `試行回数`
- `乱数シード（任意）`
- `対象週`

実行:

1. 条件を設定
2. `席替えを実行`
3. 成功したら `席替え結果` タブで確認

---

### 4-3. 席替え結果

主な用途:

- 結果確認
- 手動調整
- テーブル配置（見た目）調整
- 各種出力
- 履歴保存

#### A. 手動調整

- `table_no` を直接編集し `手動調整を反映` で確定
- 制約違反（禁止スキル同席、min/max違反など）はエラー表示

#### B. テーブル配置設定（UI表示と出力レイアウト）

`テーブル配置設定（前4・中5・後4の千鳥）` で表示位置を変更できます。

- `前列（左→右の table_no）` 最大4
- `中列（左→右の table_no）` 最大5
- `後列（左→右の table_no）` 最大4

ルール:

- 1〜テーブル数の番号を重複なしで指定
- 欠けがあるとエラー
- `配置設定を反映` で確定

反映先:

- 画面内の `テーブル別表示`
- Excel出力（通常版/反転版）
- Googleスプレッドシート出力（通常版/反転版）
- 履歴再読込後の表示

既定（13テーブル時）:

- 前列: `1,4,7,10`
- 中列: `2,5,8,11,13`
- 後列: `3,6,9,12`

#### C. 出力

- CSV: `現在の結果をCSV出力`
- Excel: `現在の結果をExcel(.xlsx)で出力（通常版+反転版）`
  - 通常版: 後ろ→前の見え方
  - 反転版: 前→後ろの見え方
- Googleスプレッドシート: `Googleスプレッドシートへ2種類を作成`

注意:

- Excel/Google出力には **スキルは出力しません**（氏名ふりがな・氏名・会社のみ）

#### D. 履歴保存

- `この結果を履歴に保存` で保存
- 保存時に条件設定と配置設定も保存

---

### 4-4. 履歴

主な用途:

- 過去結果の確認
- 過去結果CSV出力
- 過去結果を結果タブへ再読込

操作:

1. 履歴IDを選択
2. 詳細表示（前/中/後の配置で表示）
3. 必要なら `この履歴をCSV出力`
4. `この履歴を結果タブに読み込む`

## 5. Googleスプレッドシート自動出力の準備

1. Google Cloudでサービスアカウントを作成
2. JSONキーを発行してダウンロード
3. JSON内 `client_email` を対象スプレッドシートへ `編集者` で共有
4. 結果タブで以下を入力
   - 出力先スプレッドシートURL/ID
   - テンプレートシート名
   - サービスアカウントJSON
5. `Googleスプレッドシートへ2種類を作成`

補足:

- 指定テンプレート名が見つからない場合は、可能なら `座席表（テンプレ）` へフォールバックします。

## 6. 席替えロジック仕様（現行）

- 収容制約:
  - 全員を必ずどこかのテーブルに配置
  - 各テーブルは `min_per_table`〜`max_per_table`
- 禁止同席（ハード制約）:
  - `高い × 低い`
  - `高い × ヤバい`
- 優先同席（ソフト制約）:
  - 優先: `高い-並`, `並-低い`, `低い-ヤバい` の隣接スキル
  - `並 × ヤバい` は可能な限り回避（最終調整でスワップ最適化）
- スコア最適化:
  - 会社重複
  - 前回同席再発
  - スキル混在
  - 人数偏差
- 生成後、テーブル番号は代表スキル優先で再番号付けされます（`高い → 並 → 低い → ヤバい`）。

## 7. トラブルシュート

- `streamlit` が見つからない
  - 仮想環境を有効化して `python -m pip install -r requirements.txt`
  - 起動は `python -m streamlit run app.py`
- `Port 8501 is not available`
  - 既存プロセス停止、または `--server.port 8502`
- Google出力で失敗
  - JSONが正しいか
  - `client_email` が対象スプレッドシートの編集者か
  - テンプレートシート名が正しいか
